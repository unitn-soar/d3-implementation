	\section{Document Objective}
		This document aims to give a fully detailed explanation of the functionalities and development of the Soar web application for the purchase and handling of flight tickets. This document will contain the user flows for each of the four different types of user(Person, Travel Agency, Admin and Developer), provide motivations for the choices made during development and give insight in the code's functions, classes, and implementations. In the end, there will be a general team assessment, giving details on how each member of the team contributed to the project
		
		
	\section{Architecture Choice and Language}
				
		\subsection{Covered Objectives}
		\begin{enumerate}[leftmargin=2.2em,label=\textbf{O\arabic*.}]
			\begin{adjustwidth}{0.4cm}{0cm}
        		For the development of this application, the team has decided on a three tier architecture. This type of architecture provides separate functionalities for each type of user, that interact with a single back-end program that manages the various interactions with the system. The program then interacts with a separate database, containing informations related to users, flights and purchases. This architecture provides a clear separation of tasks, thus improving security on the database and making maintenance easier in case of updates or bug fixing.
                
			\end{adjustwidth}
		\end{enumerate}
		
		\subsection{Language Used}
        		\begin{enumerate}[leftmargin=2.2em,label=\textbf{O\arabic*.}]
			\begin{adjustwidth}{0.4cm}{0cm}
                For the program back-end, the language chosen was Java, because it is a versatile language made to handle object oriented programming, of which the program will make use frequently. Ideally, the program's front end would be a public website, and it would be written in HTML, but given the restriction of the project it is implemented as a private CLI. The code interacts with the database with a set of SQL queries to retrieve pertinent informations.
			\end{adjustwidth}
		\end{enumerate}
